# Документация для разработчиков (Dev Docs)

Этот документ содержит информацию о структуре проекта, настройке окружения и процессе разработки бота.

## 📋 О проекте

**Telegram Job Bot** — это бот для поиска работы и сотрудников, ориентированный на рынок Узбекистана.
Бот позволяет:
*   **Соискателям:** Создавать резюме, искать вакансии, откликаться на них.
*   **Работодателям:** Публиковать вакансии, искать сотрудников, приглашать на собеседования.
*   **Администраторам:** Управлять пользователями, делать рассылки, просматривать статистику и жалобы.

Стек технологий:
*   **Язык:** Python 3.8+
*   **Фреймворк:** `pyTelegramBotAPI` (telebot)
*   **База данных:** SQLite
*   **Тестирование:** `pytest`, `unittest.mock`

## 🚀 Установка и запуск

### 1. Клонирование репозитория
```bash
git clone <url-репозитория>
cd Bot1
```

### 2. Создание виртуального окружения
Рекомендуется использовать виртуальное окружение для изоляции зависимостей.
```bash
python -m venv venv
# Windows
venv\Scripts\activate
# Linux/macOS
source venv/bin/activate
```

### 3. Установка зависимостей
```bash
pip install -r requirements.txt
```
*Если файла `requirements.txt` нет, установите основные библиотеки:*
```bash
pip install pyTelegramBotAPI python-dotenv prometheus-client sentry-sdk
pip install pytest pytest-cov flake8 bandit  # Для разработки
```

### 4. Настройка переменных окружения
Создайте файл `.env` в корне проекта и добавьте следующие переменные:
```env
TELEGRAM_BOT_TOKEN=ваш_токен_от_BotFather
ADMIN_IDS=123456789,987654321
```

### 5. Запуск бота
```bash
python bot.py
```
При первом запуске автоматически создастся файл базы данных `jobs_database.db` и применятся миграции.

## 🧪 Тестирование

Проект имеет высокое покрытие тестами (>90%). Используется `pytest`.

```bash
pytest

pytest --cov=. --cov-report=term-missing

**Линтинг и проверка безопасности:**

```

## 🛠 Разработка

### Добавление нового функционала
1.  **Хендлеры:** Создайте новый метод в соответствующем файле в `handlers/` или создайте новый файл, если логика изолирована.
2.  **Роутинг:** Зарегистрируйте хендлер в `bot.py` или добавьте его вызов в существующие диспетчеры (например, в `process_all_messages`).
3.  **БД:** Если нужны новые таблицы или поля, добавьте их в `database/schema.py` (функция `init_database` или миграции).
4.  **Тесты:** Обязательно напишите тесты в `tests/`.

### Машина состояний (FSM)
Для пошаговых сценариев (регистрация, создание вакансии) используется кастомная FSM на базе SQLite.
*   `database.set_user_state(user_id, state_dict)` — установить состояние.
*   `database.get_user_state(user_id)` — получить текущее состояние.
*   `handlers/steps.py` — центральный обработчик шагов.
