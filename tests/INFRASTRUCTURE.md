# Инфраструктурная документация

Этот документ описывает архитектуру, развертывание и обслуживание бота.

## 🏗 Архитектура

Проект представляет собой монолитное Python-приложение.

*   **Runtime:** Python 3.8+
*   **База данных:** SQLite (файл `jobs_database.db`). Используется для хранения пользователей, вакансий, откликов и состояний FSM.
*   **Контейнеризация:** Docker.

### Схема работы
1.  Бот запускается и инициализирует БД (применяет миграции).
2.  Устанавливает соединение с Telegram (Long Polling).
3.  Запускает HTTP-сервер для метрик (порт 8000).
4.  Обрабатывает входящие сообщения в многопоточном режиме (ограничено GIL, но `telebot` использует трединг).

## 🐳 Развертывание (Docker)

Необходимо установить Docker.

Для развертывания используется `docker compose` (V2) или `docker-compose` (V1).

### Структура `docker-compose.yml`
```yaml
version: '3.8'
services:
  bot:
    build: .
    restart: always
    env_file: .env
    volumes:
      - ./jobs_database.db:/app/jobs_database.db  # Персистентность БД
      - ./backups:/app/backups                    # Папка для бэкапов
      - ./bot.json.log:/app/bot.json.log          # Логи
    ports:
      - "8000:8000"                               # Метрики
```

### Команды для деплоя
```bash
# Сборка и запуск
docker compose up -d --build

# Просмотр логов
docker compose logs -f

# Остановка
docker compose down
```

## ⚙️ Конфигурация (.env)

| Переменная | Обязательно | Описание |
| :--- | :--- | :--- |
| `TELEGRAM_BOT_TOKEN` | Да | Токен бота от @BotFather |
| `ADMIN_IDS` | Да | Список ID администраторов через запятую (напр. `123,456`) |
| `SENTRY_DSN` | Нет | DSN для интеграции с Sentry |
| `PROMETHEUS_PORT` | Нет | Порт для метрик (по умолчанию 8000) |

## 🔄 CI/CD (GitHub Actions)

В проекте настроен workflow `.github/workflows/ci.yml`.

### Этапы (Jobs):
1.  **Build & Test:**
    *   Запуск на Python 3.8, 3.9, 3.10.
    *   Линтинг (`flake8`).
    *   Проверка безопасности (`bandit`).
    *   Запуск тестов (`pytest`) с проверкой покрытия.
2.  **Deploy:**
    *   Запускается только при пуше в ветку `main`.
    *   Подключается к серверу по SSH.
    *   Обновляет код (`git pull`).
    *   Пересобирает и перезапускает Docker-контейнер.

## 💾 Резервное копирование

*   **База данных:** SQLite файл `jobs_database.db`.
*   **Механизм:** Встроенная команда `/backup` (доступна админам) создает копию БД в папку `backups/` и отправляет файл в чат.
*   **Рекомендация:** Настроить регулярное копирование файла `jobs_database.db` средствами хост-системы (например, cron + rsync).

## 📈 Мониторинг

Рекомендуемый стек: **Prometheus + Grafana**.
1.  Настройте Prometheus на сбор метрик с порта 8000.
2.  Импортируйте дашборд в Grafana для визуализации RPS, ошибок и задержек.
